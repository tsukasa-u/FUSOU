# 通信安全基準（Communication Integrity）

## 1. 目的

本基準は、通信を扱う機能において、利用者・権利者・サービス運営の安全性を維持するための運用指針です。

## 2. 設計原則

本サービスの通信関連機能は、次の原則に基づいて設計・運用します。

- 観測専用: 通信の読み取りを目的とし、改変を行わない。
- 非干渉: 外部サービスへ不要な追加通信を送信しない。
- 非偽装: パケット改変・スプーフィング・なりすましを行わない。
- 非自動化: チート・マクロ・自動操作を提供しない。
- 最小権限: 必要最小限の権限で機能を実装する。

## 3. 運用時の確認項目

- 新機能追加時に、通信改変の有無をレビューする。
- 外部 API 利用時に、送信データの最小化を確認する。
- 障害解析ログは、個人情報・機密情報の露出を避けて保持する。

## 4. インシデント対応

問題が確認された場合は、次を優先して対応します。

- 影響範囲の特定
- 機能停止または制限
- 修正と再発防止策の実施
- 必要に応じた告知
